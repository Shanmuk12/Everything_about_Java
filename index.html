JVM Architecture & Memory Model

Heap · Metaspace · Stack · TLAB · G1GC · Young/Old Gen · Tenuring
DEEP ▼

Core Concept

  • Classloader subsystem: Bootstrap → Extension → App
  • Heap: Young Gen (Eden + S0 + S1) → Old Gen → Metaspace
  • TLAB: Thread-Local Allocation Buffer — allocations without CAS
  • JIT: C1 (quick) → C2 (optimized) compilation tiers
  • G1GC: Region-based, concurrent, low-pause collector

Interview Traps

  • "Where does String pool live?" — Heap since Java 7
  • OutOfMemoryError in Metaspace ≠ Heap OOM
  • Stack overflow ≠ Heap overflow — different regions
  • Static variables in Metaspace, not Stack
  • TLAB exhaustion causes minor GC, not major

Microservices Connection

  • Tune -Xmx / -Xms to avoid container OOMKill
  • G1GC for low-latency REST APIs (-XX:MaxGCPauseMillis=100)
  • Metaspace explosion with dynamic classloading (Groovy scripts)
  • ZGC / Shenandoah for sub-millisecond pauses in Kafka consumers
  • Monitor via JFR + Micrometer in Spring Boot Actuator

Production Debug Scenario

  • Problem: Spring Boot pod OOMKilled every 6 hours
  • Root cause: Heap leak in ThreadLocal not removed after request
  • Fix: Use try-finally in Filter to call remove()
  • Tool: jmap -histo:live <pid> + Eclipse MAT analysis

Object Creation & Lifecycle

new keyword flow · Object header · Mark word · Escape analysis
DEEP ▼

Object Creation Flow

  • 1. Check class is loaded → 2. Allocate in TLAB → 3. Zero-initialize fields → 4. Set object header → 5. Run constructor
  • Object header: mark word (8B) + klass pointer (4B compressed)
  • Mark word stores: hashCode, GC age bits, lock state, monitor ptr
  • Escape analysis: JVM can stack-allocate if object doesn't escape method

Internals Deep Dive

  • Age bits: 4 bits → max tenuring threshold = 15
  • Biased locking removed in JDK 15 (JEP 374)
  • Virtual threads (JDK 21): object on heap, stack in heap
  • Records: auto-generate compact constructor + final fields
  • Singleton: double-checked locking needs volatile for safe publication

Interview Questions

  • "How is new String() different from literal?"
  • "When does an object get promoted to Old Gen?"
  • "Can Java allocate on stack?" — Yes, escape analysis
  • "What's inside an object header?"
  • "Why is double-checked singleton broken without volatile?"

String Deep Dive

Pool · Immutability · + operator bytecode · split/substring internals
DEEP ▼

Immutability Internals

  • char[] value is private final — but array itself is mutable (reflection risk)
  • Java 9+: compact strings — LATIN1 uses byte[], UTF16 uses byte[2*n]
  • String pool: HashMap<String,WeakRef> in heap (interning)
  • intern() manually adds to pool — use for memory optimization
  • s1 == s2 for literals (same pool ref); .equals() for value comparison

+ Operator Bytecode

  • Pre Java 9: compiled to StringBuilder().append().toString()
  • Java 9+: invokedynamic via StringConcatFactory (faster)
  • In loops: each iteration creates new StringBuilder — use explicit SB
  • StringBuilder: not thread-safe, O(1) amortized append
  • StringBuffer: synchronized, use only when shared across threads

split() / substring() Source

  • split(regex): compiles Pattern every call — cache Pattern for perf
  • Java 7u6+: substring() copies char[] (fixed memory leak from pre-7 view)
  • indexOf() uses naive O(n*m) for short patterns, Boyer-Moore for longer
  • String.format() is ~5x slower than concatenation — avoid in hot paths

DSA Critical Patterns

  • char[] over String for O(1) mutation in problems
  • toCharArray() = O(n) copy — do once, then work on array
  • s.charAt(i) - 'a' → 0-indexed alphabet mapping
  • Integer.parseInt() throws NumberFormatException — always validate

Primitive Types & Wrappers

Autoboxing traps · Integer cache · Memory layout
MED ▼

Autoboxing Traps

  • Integer.valueOf(-128 to 127) returns cached instance
  • Integer a=127; Integer b=127; a==b → TRUE (cached)
  • Integer a=128; Integer b=128; a==b → FALSE (new objects!)
  • Long l = null; long primitive = l; → NullPointerException!
  • List<Integer> remove(int index) vs remove(Integer object) — autoboxing matters

Memory Layout

  • int: 4B on stack / in object, Integer: 16B on heap
  • int[] arr = new int[n]: contiguous 4n bytes
  • Integer[] arr: array of 8B pointers + objects scattered
  • Use int[] in DSA — Integer[] causes boxing overhead
  • HashMap<Integer,Integer> = ~48B per entry overhead
⚠ Interview Trap: Comparing Integer with == instead of .equals() is one of the most common Java bugs. Always use .equals() for object comparison, primitives use ==.

HashMap Deep Dive

  • hash(key) = key.hashCode() ^ (h >>> 16) — mixes high bits
  • index = (n-1) & hash — why n must be power of 2
  • Java 8+: bucket with >=8 entries → treeify to RB tree
  • Java 8+: resize doubles table; re-hashes all entries
  • ConcurrentHashMap: 16 segment locks → one lock per bin (Java 8+)

Fail-Fast vs Fail-Safe

  • ArrayList/HashMap iterators: fail-fast (modCount check → CME)
  • CopyOnWriteArrayList: fail-safe (iterates snapshot copy)
  • ConcurrentHashMap iterator: weakly consistent, no CME
  • Use removeIf() or Iterator.remove() to avoid CME

Concurrency for DSA & Microservices

volatile · synchronized · ReentrantLock · CompletableFuture · Virtual Threads JDK 21
CRITICAL ▼

Java Memory Model Basics

  • happens-before: write X then read X in different thread
  • volatile: visibility guarantee + prevents reordering; no atomicity
  • synchronized: mutual exclusion + happens-before guarantee
  • AtomicInteger: CAS loop, lock-free, better than sync for counters
  • StampedLock: optimistic read, better than RW lock for read-heavy

Virtual Threads (JDK 21)

  • Thread.ofVirtual().start(() -> task) — millions of VT possible
  • VT pins to carrier thread on synchronized block — use ReentrantLock
  • Perfect for I/O-bound microservices: Spring Boot 3.2 + VT
  • ThreadLocal works, but InheritableTL has issues with VT pooling
  • Structured concurrency: StructuredTaskScope (JDK 21 preview)

CompletableFuture Patterns

  • thenCompose: sequential async (flatMap for futures)
  • thenCombine: parallel async, combine two results
  • allOf: wait for all; anyOf: first to complete
  • exceptionally: error handling in async chain
  • Use custom Executor, never default ForkJoinPool for blocking ops

Lambdas & Streams Internals

Functional interfaces · Lazy evaluation · Parallel streams · Spliterator
MED ▼

Lambda Internals

  • Lambdas compile to invokedynamic (LambdaMetafactory at runtime)
  • Non-capturing lambda: single instance (stateless singleton)
  • Capturing lambda: creates new object per call — watch in loops
  • Method refs are faster than lambdas in some JVM implementations
  • @FunctionalInterface enforces exactly one abstract method

Stream Pipeline

  • Lazy: intermediate ops build pipeline, terminal triggers execution
  • Short-circuit: findFirst(), anyMatch() stop early
  • Stream reuse: IllegalStateException after terminal op
  • Spliterator: enables parallel splitting of source data
  • parallel(): uses ForkJoinPool.commonPool() — be careful with blocking ops

DSA with Streams (Interview Style)

  • Collectors.groupingBy() for frequency maps
  • Collectors.toMap() for index-to-value maps
  • IntStream.range(0,n) for index-based loops
  • Stream.iterate() for infinite sequence generation

Binary Search (All Variants)

Standard · Left/Right bound · On answer · Custom comparators
MED ▼

3 Templates to Master

  • T1 — exact search: while(l<=r) mid=(l+r)/2
  • T2 — left bound: while(l<r) shrink right when valid
  • T3 — right bound: while(l<r) shrink left when valid
  • Binary search on answer: can(mid) check function pattern

Java Specific

  • Collections.binarySearch(list, key) — same semantics
  • TreeMap.floorKey(k) / ceilingKey(k) = binary search on BST
  • Custom: Arrays.sort(arr, Comparator) + binarySearch with same Comparator

Graphs — BFS · DFS · Dijkstra · Union-Find

Adjacency list · Topological sort · Path compression · Rank union
HARD ▼

Java Graph Representation

  • Adjacency list: List<List<int[]>> adj = new ArrayList()
  • Weighted: int[][] edges or List<int[]> per node
  • BFS: ArrayDeque (faster than LinkedList) as queue
  • DFS: recursive (watch stack overflow for n>10^4) or iterative
  • Visited: boolean[] vis or int[] dist initialized to -1

Dijkstra in Java

  • PriorityQueue<int[]> pq sorted by distance
  • Always check if dist[node] < current to skip stale entries
  • O((V+E) log V) with binary heap PQ
  • Negative weights → Bellman-Ford (no Dijkstra!)

Union-Find (DSU)

  • Path compression: parent[x] = find(parent[x]) in find()
  • Union by rank: attach smaller rank tree under larger
  • Nearly O(α(n)) ≈ O(1) per operation
  • Track connected components count with a variable

Dynamic Programming

1D · 2D · State transitions · Space optimization · Bitmask DP
HARD ▼

DP Framework

  • 1. Define state: dp[i] = "optimal value considering first i items"
  • 2. Recurrence: dp[i] = f(dp[i-1], dp[i-2], ...)
  • 3. Base case: dp[0] or dp[1]
  • 4. Order: bottom-up (tabulation) preferred in Java — no stack overflow
  • 5. Space optimize: keep only last 1 or 2 rows when possible

Java-Specific DP

  • int vs long: use long when n > 10^4 and accumulating sums
  • Memoization: HashMap<String, Long> for complex states
  • For Bitmask DP: int[] dp = new int[1<<n]
  • 2D array: int[][] dp = new int[n+1][m+1] — declare once
  • Arrays.fill() for initialization; careful with Integer.MIN_VALUE+1

static class FastReader { BufferedReader br = new BufferedReader(new InputStreamReader(System.in)); StringTokenizer st; String next() { while (st == null || !st.hasMoreTokens()) { try { st = new StringTokenizer(br.readLine()); } catch (IOException e) { throw new RuntimeException(e); } } return st.nextToken(); } int nextInt() { return Integer.parseInt(next()); } long nextLong() { return Long.parseLong(next()); } String nextLine() { try { return br.readLine(); } catch (IOException e) { throw new RuntimeException(e); } } }
DSU.java — Union-Find with Path Compression + Rank
static int[] parent, rank; static int components; static void init(int n) { parent = new int[n]; rank = new int[n]; components = n; for (int i = 0; i < n; i++) parent[i] = i; } static int find(int x) { if (parent[x] != x) parent[x] = find(parent[x]); // path compression return parent[x]; } static boolean union(int a, int b) { int ra = find(a), rb = find(b); if (ra == rb) return false; if (rank[ra] < rank[rb]) { int t = ra; ra = rb; rb = t; } parent[rb] = ra; if (rank[ra] == rank[rb]) rank[ra]++; components--; return true; }
FenwickTree.java — Binary Indexed Tree (BIT)
static int[] bit; static int N; static void update(int i, int delta) { // 1-indexed for (; i <= N; i += i & (-i)) bit[i] += delta; } static int query(int i) { // prefix sum [1..i] int s = 0; for (; i > 0; i -= i & (-i)) s += bit[i]; return s; } static int range(int l, int r) { // sum [l..r] return query(r) - query(l - 1); }
SegTree.java — Segment Tree (Range Sum + Point Update)
static long[] tree; static int N; static void build(int[] arr, int node, int start, int end) { if (start == end) { tree[node] = arr[start]; return; } int mid = (start + end) / 2; build(arr, 2*node, start, mid); build(arr, 2*node+1, mid+1, end); tree[node] = tree[2*node] + tree[2*node+1]; } static void update(int node, int start, int end, int idx, int val) { if (start == end) { tree[node] = val; return; } int mid = (start + end) / 2; if (idx <= mid) update(2*node, start, mid, idx, val); else update(2*node+1, mid+1, end, idx, val); tree[node] = tree[2*node] + tree[2*node+1]; } static long query(int node, int start, int end, int l, int r) { if (r < start || end < l) return 0; if (l <= start && end <= r) return tree[node]; int mid = (start + end) / 2; return query(2*node, start, mid, l, r) + query(2*node+1, mid+1, end, l, r); }
Dijkstra.java — Shortest Path Template
static int[] dijkstra(List<int[]>[] graph, int src, int n) { int[] dist = new int[n]; Arrays.fill(dist, Integer.MAX_VALUE); dist[src] = 0; // PQ: [distance, node] PriorityQueue<int[]> pq = new PriorityQueue<>((a,b) -> a[0]-b[0]); pq.offer(new int[]{0, src}); while (!pq.isEmpty()) { int[] curr = pq.poll(); int d = curr[0], u = curr[1]; if (d > dist[u]) continue; // skip stale entry for (int[] e : graph[u]) { int v = e[0], w = e[1]; if (dist[u] + w < dist[v]) { dist[v] = dist[u] + w; pq.offer(new int[]{dist[v], v}); } } } return dist; }
